Giai đoạn 1: Offline Setup (Trusted Center)

TC tạo Khóa chủ (Master Secret) $S$ và nạp các Token dài hạn $T_A$ cho các thiết bị IoT trước khi ngắt mạng (offline).

Khóa chủ (S)
Chưa tạo
Alice ID ($ID_A$)
Bob ID ($ID_B$)
Public Key ($pk_A$)
Token dài hạn ($T_A$)
H(S || pk_A || ID_A)

Giai đoạn 2: Initiation (Alice → Bob)

Alice tạo nonce $r_A$ và tính toán Token ngắn hạn $t_A$ để chứng minh danh tính mà không cần gửi Khóa chủ.

Nonce ($r_A$)
Token ngắn hạn ($t_A$)
H(T_A || r_A)
Message 1 ($M_1$)

Giai đoạn 3: Encapsulation (Bob → Alice)

Bob kiểm tra $t_A$ trước (Chống DoS). Nếu hợp lệ, chạy ML-KEM Encapsulation và sinh mã MAC để phản hồi.

Token kiểm tra ($t'_A$)
Trạng thái Token
Ciphertext ($ct$)
Shared Secret ($ss$)
Khóa phiên ($K$)
KDF(ss || S || r_A || r_B)
Mã xác thực ($MAC_B$)
Message 2 ($M_2$)

Giai đoạn 4: Verification (Alice)

Alice decapsulate để lấy $ss$, tự tính lại Khóa phiên $K$ và kiểm tra mã MAC của Bob.

Khôi phục $ss$

Lưu ý: Các thuật toán KEM.Encaps và KEM.Decaps được mô phỏng ngẫu nhiên chuỗi byte để đảm bảo Web nhẹ. Hashing (SHA-256) và MAC (HMAC) được chạy thực tế bằng Web Crypto API.
