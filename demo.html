TB-PQ-AKE Protocol Execution

Functional demonstration of the Token-Based Post-Quantum Authenticated Key Exchange.
Cryptographic hashes and MACs are live (Web Crypto API). ML-KEM operations are simulated.

Phase 1: Offline System Setup (Trusted Center)

The Trusted Center generates the Master Secret Key (S), KEM keypairs, and Long-term Tokens.

Phase 2: Initiation (Alice → Bob)

Alice generates an ephemeral nonce and creates a Short-term Token to prove her identity.

Phase 3: Encapsulation & Response (Bob → Alice)

Bob verifies the token first (Authenticate First, Compute Later). If valid, he runs KEM encapsulation.

Phase 4: Decapsulation & Verification (Alice)

Alice decapsulates the shared secret and verifies Bob's MAC to achieve mutual authentication.
